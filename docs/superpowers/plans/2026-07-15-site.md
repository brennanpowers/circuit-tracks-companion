# Circuit Tracks Companion Site (Phase 3) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and deploy the public learning site — a project-driven course, a wiki reference derived *directly* from `guide/`, an interactive panel map, and a printable cheat sheet — on GitHub Pages.

**Architecture:** Astro 5 static site in `site/`. The reference section loads `../guide/*.md` straight through Astro's content-layer glob loader (single source of truth — no copied content), with a remark plugin rewriting inter-guide links to site routes. Course lessons are MDX with typed frontmatter and reusable callout components. Interactivity (theme, progress, panel map, search) is vanilla `<script>` islands — no UI framework. CI builds, type-checks, link-checks, and deploys on green.

**Tech Stack:** Astro ^5, @astrojs/mdx, Tailwind CSS ^4 (@tailwindcss/vite), Pagefind ^1 (search), Vitest ^3 (unit tests), @fontsource-variable/inter + @fontsource-variable/jetbrains-mono, GitHub Actions → GitHub Pages.

## Global Constraints

- **Tracks only; guide is canonical.** Site content derives from `guide/` and may simplify/reorder but never contradict it. Reference pages ARE the guide files (loaded, not copied).
- **Accurate · functional · modern-and-pretty** (spec success criteria). CI blocks deploy on `astro check`, build, or link-check failure.
- **Mobile is a first-class target** (the screen next to the device). Dark theme default, light optional, `prefers-reduced-motion` respected, keyboard navigable.
- **No accounts, comments, analytics, or hosted media** (spec: out of scope). Progress = `localStorage` only.
- **GitHub Pages project site:** `site: 'https://brennanpowers.github.io'`, `base: '/circuit-tracks-companion'`. Every internal href must go through `withBase()` or the remark rewriter — never hardcode `/course/...`.
- **Combo notation** matches the guide: backticked `` `Shift + Play` `` with capitalized button names.
- **Device Labs:** the four `[device-check]` items from `research/open-questions.md` appear as `<DeviceLab>` callouts in the specific lessons noted in Tasks 8–10; results flow back to `research/` and `guide/`, not just the site.
- Commit messages: imperative, no AI references (repo rule).

```ts
import type { Root, Link } from 'mdast';
import { visit } from 'unist-util-visit';

export function rewriteGuideHref(href: string, base: string): string | null {
  if (/^(https?:)?\/\//.test(href) || href.startsWith('#')) return null;
  const m = href.match(/^(?:\.\/)?(README|\d\d-[a-z0-9-]+)\.md(#.*)?$/);
  if (!m) return null;
  const [, stem, anchor = ''] = m;
  const path = stem === 'README' ? 'reference/' : `reference/${stem}/`;
  return `${base.replace(/\/$/, '')}/${path}${anchor}`;
}

export function remarkGuideLinks() {
  return (tree: Root, file: { path?: string }) => {
    // Only rewrite links inside files loaded from guide/
    if (!file.path?.includes('/guide/')) return;
    const base = process.env.SITE_BASE ?? '/circuit-tracks-companion';
    visit(tree, 'link', (node: Link) => {
      const out = rewriteGuideHref(node.url, base);
      if (out) node.url = out;
    });
  };
}
```

### Task 5: Link checker script

**Files:**
- Create: `site/scripts/check-links.mjs`

**Interfaces:**
- Produces: `npm run test:links` — exits 1 listing broken internal hrefs/srcs across `dist/**/*.html`.

- [ ] **Step 1: Write the script:**

```js
import { readdirSync, readFileSync, existsSync, statSync } from 'node:fs';
import { join, dirname, resolve } from 'node:path';

const DIST = resolve('dist');
const BASE = '/circuit-tracks-companion';
const htmlFiles = [];
(function walk(d) {
  for (const f of readdirSync(d)) {
    const p = join(d, f);
    statSync(p).isDirectory() ? walk(p) : f.endsWith('.html') && htmlFiles.push(p);
  }
})(DIST);

const broken = [];
for (const file of htmlFiles) {
  const html = readFileSync(file, 'utf8');
  for (const [, href] of html.matchAll(/(?:href|src)="([^"#?]+)[^"]*"/g)) {
    if (/^(https?:|mailto:|data:)/.test(href)) continue;
    let target = href.startsWith(BASE) ? href.slice(BASE.length) : href;
    let fsPath = target.startsWith('/') ? join(DIST, target) : join(dirname(file), target);
    if (fsPath.endsWith('/')) fsPath = join(fsPath, 'index.html');
    if (!existsSync(fsPath) && !existsSync(fsPath + '.html') && !existsSync(join(fsPath, 'index.html')))
      broken.push(`${file.replace(DIST, '')} -> ${href}`);
  }
}
if (broken.length) { console.error('BROKEN LINKS:\n' + broken.join('\n')); process.exit(1); }
console.log(`OK: ${htmlFiles.length} pages, no broken internal links`);
```

- [ ] **Step 4: Implement `progress.ts`:**

```ts
const KEY = 'ctc-progress';
export function getDone(): string[] {
  try { const v = JSON.parse(localStorage.getItem(KEY) ?? '[]'); return Array.isArray(v) ? v : []; }
  catch { return []; }
}
export const isDone = (id: string) => getDone().includes(id);
export function toggle(id: string): void {
  const done = new Set(getDone());
  done.has(id) ? done.delete(id) : done.add(id);
  localStorage.setItem(KEY, JSON.stringify([...done]));
  document.dispatchEvent(new CustomEvent('ctc-progress-change'));
}
export const percent = (ids: string[]) =>
  ids.length ? Math.round((ids.filter(isDone).length / ids.length) * 100) : 0;
```

```yaml
name: Deploy site
on:
  push: { branches: [main] }
  workflow_dispatch:
permissions: { contents: read, pages: write, id-token: write }
concurrency: { group: pages, cancel-in-progress: true }
jobs:
  build:
    runs-on: ubuntu-latest
    defaults: { run: { working-directory: site } }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 22, cache: npm, cache-dependency-path: site/package-lock.json }
      - run: npm ci
      - run: npm run check
      - run: npm test
      - run: npm run build
      - run: npm run test:links
      - uses: actions/upload-pages-artifact@v3
        with: { path: site/dist }
  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment: { name: github-pages, url: ${{ steps.deployment.outputs.page_url }} }
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4
```
